# Input Field Policy 輸入欄位共用政策

## 政策定位

- 本 module 內所有含文字或金額輸入的 editor 類 screen 共用政策
- 統一輸入欄位的邊界行為：達上限阻擋、空白去除、查重與存檔驗證分工
- screen spec 只標引用，不重述邊界行為細節
- 長度與位數的具體字數由對側 Model git 的欄位長度約束承載，本政策只以語意引用、不列數字
- 存檔時的驗證失敗與失敗原因由 no3_logics 對應操作承載
- 視覺定案由對側 Design git 的 `10_foundations/` 與 `20_components/` 承載
- 實作元件、平台 API、套件選用由對側 Impl git 的 `CLAUDE.md` UI Coding Guideline 承載

---

## 適用欄位

- 名稱輸入框
    - 帳戶名稱
    - 分類名稱
- 備註輸入框
    - 交易備註
    - 轉帳備註
    - 排程 template 備註
- 金額輸入框
    - 交易金額
    - 轉帳的轉出、轉入金額

---

## 長度與位數即時阻擋

- 長度上限與金額位數上限採即時阻擋
- **IF** 輸入已達上限:
    - 後續輸入不再更新欄位內容
    - 不顯示錯誤訊息，靜默阻擋
- 各欄位的長度與位數上限依 Model 層欄位長度約束
    - 名稱長度上限依 Model 層名稱欄位長度約束
    - 備註長度上限依 Model 層備註欄位長度約束
    - 金額位數上限依 Model 層金額儲存上界推得，確保縮放後不超過系統安全整數上界

---

## 空白去除與必填判定

- 名稱與備註於寫入時去除前後空白
- **IF** 名稱去除前後空白後為空:
    - 視為空
    - 由完成按鈕的必填閘擋下
- 全部為空白的名稱等同未填，不另立空白名稱

---

## 查重政策

- 不做查重
- 允許同名帳戶、同名分類並存
- 畫面不顯示撞名警告
- 不因撞名將完成按鈕設為 disabled

---

## 金額存檔驗證

- 金額超出可儲存範圍屬存檔時的驗證失敗，非即時阻擋
    - 即時位數上限已於輸入端約束，此為繞過畫面路徑的後備
    - 存檔驗證失敗原因為金額超出範圍
- 名稱超過長度上限屬存檔時的驗證失敗，作為畫面即時上限的後備
    - 存檔驗證失敗原因為名稱過長
- 匯入路徑的名稱超長不作驗證失敗，截斷至長度上限後存入
    - 與畫面端即時上限的效果等價，匯入以保留資料列為優先
- 金額門檻依各 editor：交易金額不得為 0；轉帳的轉出、轉入金額皆須大於 0
    - 門檻的驗證失敗由 no3_logics 對應操作承載

---

## 完成按鈕 disabled 條件

- disabled 條件維持既有，不因查重或長度新增條件
    - 必填欄位未填
    - 金額門檻未達（金額須大於 0 或不等於 0，依各 editor）
    - 轉帳的轉出與轉入帳戶相同
- 長度超限靠即時阻擋處理，不進 disabled 條件
- 金額超限靠存檔驗證處理，不進 disabled 條件

---

## screen spec 引用方式

- **要求:**
    - 各 screen spec 在對應輸入欄位標註 `依輸入欄位共用政策`
    - 需即時上限行為時，以行為語意描述達上限即時阻擋，不列具體字數
    - 不重述空白去除與查重規則
    - 不重述完成按鈕 disabled 條件的完整清單

- **合規寫法:**

```
- 備註輸入框
  - 依輸入欄位共用政策
  - 達備註長度上限即時阻擋，輸入不再更新
```
