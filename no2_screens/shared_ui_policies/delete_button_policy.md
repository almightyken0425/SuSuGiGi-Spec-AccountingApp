# Delete Button Policy 刪除按鈕共用政策

## 政策定位

- 本 module 內所有 editor 類 screen 共用的刪除按鈕政策
- 統一刪除按鈕的標籤與觸發行為
- screen spec 只標引用，不重述標籤與視覺
- 視覺定案由對側 Design git 的 10_foundations 與 20_components 承載
- 實作由對側 Impl git 的共用刪除元件承載

---

## 標籤詞庫

- 強制標籤一律為 刪除
- 禁止依 screen 改寫為不同字樣
- 禁止帶實體名稱，例如 刪除類別 或 刪除帳戶
- 同一元件放在不同 screen 觸發不同刪除操作，標籤不隨之改變

---

## 適用 screen

- 交易編輯器
- 轉帳編輯器
- 帳戶編輯器
- 類別編輯器

---

## 不適用情形

- 資料管理的清空資料庫為整庫重設，非單筆刪除，標籤與行為各自定義
- 合併編輯器的來源刪除為合併副作用，無獨立刪除按鈕

---

## 顯示與觸發

- 置於 screen 的操作區
- **IF** 編輯模式:
  - 顯示刪除按鈕
- 點按後觸發該 screen 對應的刪除操作
- 確認對話框與復原處理依各 screen 互動章節
- 定期交易與轉帳的刪除模式對話框分流屬各 screen 互動章節

---

## screen spec 引用方式

- 各 screen spec 在操作區以 刪除 按鈕 列出元件
- 於該元件下標註 標籤依 delete_button_policy 規範
- 不重述標籤詞庫
- 不重述視覺定案
